This is a common HTML error, but it can be tricky to debug if you're not familiar with how HTML parses.

The issue is with the nested

tags. HTML implicitly closes tags when it encounters a closing tag for a parent element. In this case, when the parser hits the closing , it implicitly closes the inner

tag. However, this leads to unexpected behaviour because the remaining text is treated as part of the outer

, which is not semantically correct.

The issue arises when you have nested

tags and you intend to have separate paragraphs within a container element.

Here's what happens:

  • The browser correctly interprets <div><p>This is some text.</p>
  • Then it encounters <p>This is more text...</p>. The closing </div> causes the implicit closing of the second <p>.
  • The result is that This is more text. This is spanned text is treated as part of the first

    tag which usually is not the intended behavior.
